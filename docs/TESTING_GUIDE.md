## 4. 시작/도착 노드 설정 위자드

> **v2 설계:** 시작/도착 노드의 전체 위자드(카테고리 → 서비스 → 요구사항 → 인증)가 ServiceSelectionPanel 오버레이 내부에서 완료된다. OutputPanel은 사용하지 않는다.

### 4.1 카테고리 선택 (Step 1)

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 4.1.1 | placeholder 클릭 | 캔버스 중앙에 카테고리 선택 오버레이가 표시됨 | |
| 4.1.2 | "어디에서 어디로 갈까요?" 제목 | 오버레이 상단에 제목이 표시됨 | |
| 4.1.3 | 검색 입력 | 검색어에 맞는 카테고리만 필터링됨 | |
| 4.1.4 | 서비스 없는 카테고리 + 요구사항 없음 (processing 등) 클릭 | 노드가 바로 배치되고 오버레이가 닫힘, isConfigured: true | |
| 4.1.5 | 서비스 있는 카테고리 클릭 (storage 등) | Step 2(서비스 선택)로 전환됨 | |
| 4.1.6 | 서비스 없는 카테고리 + 요구사항 있음 (web-scraping) 클릭 | 노드가 배치되고 오버레이 내에서 Step 3(요구사항)로 전환됨 | |

### 4.2 서비스 선택 (Step 2 — 오버레이 내부)

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 4.2.1 | 서비스 목록 표시 | 선택한 카테고리의 서비스들이 그리드로 표시됨 | |
| 4.2.2 | 왼쪽에 카테고리 아이콘 표시 | 선택한 카테고리 아이콘 + 라벨이 왼쪽에 보임 | |
| 4.2.3 | "뒤로" 클릭 | Step 1(카테고리 선택)으로 돌아감 | |
| 4.2.4 | 서비스 클릭 (예: Google Drive) | 노드가 캔버스에 배치되고, 오버레이 내에서 Step 3(요구사항)으로 전환됨 | |
| 4.2.5 | 배치된 노드 상태 | isConfigured: false (아직 위자드 미완료) | |

### 4.3 요구사항 선택 (Step 3 — 오버레이 내부)

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 4.3.1 | 요구사항 목록 표시 | 오버레이 내에서 해당 카테고리의 요구사항 옵션들이 나열됨 | |
| 4.3.2 | "뒤로" 클릭 (서비스 거쳐 옴) | 노드가 캔버스에서 제거되고, Step 2(서비스 선택)로 돌아감 | |
| 4.3.3 | "뒤로" 클릭 (서비스 없이 옴, web-scraping) | 노드가 제거되고 Step 1(카테고리 선택)으로 돌아감 | |
| 4.3.4 | InputPanel/OutputPanel 상태 | 오버레이가 열려 있으므로 양쪽 패널 모두 숨겨져 있음 | |

### 4.4 인증 (Step 4 — 오버레이 내부, requiresAuth인 경우)

| # | 테스트 | 기대 결과 | 결과 |

| 4.5.2 | 요구사항 선택 | 인증 단계 없이 updateNodeConfig → isConfigured: true, 오버레이 닫힘 | |

### 4.6 위자드 비정상 종료

> **v2 설계:** 중간 노드는 ServiceSelectionPanel에서 카테고리 선택만 하고, 이후 설정은 듀얼 패널(InputPanel + OutputPanel)에서 진행한다.

| 5.1.2 | 카테고리 선택 | 노드 배치 → 오버레이 닫힘 → 듀얼 패널 자동 열림 | |
| 5.1.3 | 오버레이에서 서비스/요구사항/인증 단계 | 나타나지 않음 (카테고리 선택 즉시 종료) | |

### 5.2 중간 노드 듀얼 패널

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 5.2.1 | 중간 노드 클릭 | 왼쪽 InputPanel + 오른쪽 OutputPanel이 동시에 열림 | |
| 5.2.2 | InputPanel 헤더 | "들어오는 데이터" + 소스 노드 아이콘이 표시됨 | |
| 5.2.3 | InputPanel 내용 | 소스 노드의 라벨, 출력 타입이 표시됨 | |
| 5.2.4 | OutputPanel (isConfigured: false + 요구사항 있음) | 요구사항 목록이 표시됨 | |

| 5.2.7 | OutputPanel (요구사항 없는 노드) | 바로 PanelRenderer가 표시됨 | |

## 테스트 진행 방법

1. **순서대로 진행**: 2장(캔버스) → 3장(Placeholder) → 4장(위자드) → 5장(중간 노드) → 6장(삭제) → 7장(닫기 규칙) → 8장(인터랙션) → 9장(Edge) → 10장(툴바) → 11장(라우팅)
2. **결과 기록**: 각 항목의 "결과" 칸에 `✅ Pass` / `❌ Fail (설명)` 기록
3. **Fail 발견 시**: 즉시 스크린샷 + 콘솔 에러 캡처
4. **매 섹션 완료 후**: 페이지 새로고침하여 상태 초기화 후 다음 섹션 진행
