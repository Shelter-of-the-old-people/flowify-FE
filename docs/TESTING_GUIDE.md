## 4. 시작/도착 노드 설정 위자드

### 4.1 카테고리 선택 (Step 1)

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 4.1.1 | placeholder 클릭 | 캔버스 중앙에 카테고리 선택 오버레이가 표시됨 | |
| 4.1.2 | "어디에서 어디로 갈까요?" 제목 | 오버레이 상단에 제목이 표시됨 | |
| 4.1.3 | 검색 입력 | 검색어에 맞는 카테고리만 필터링됨 | |
| 4.1.4 | 서비스 없는 카테고리 + 요구사항 없음 (processing 등) 클릭 | 노드가 바로 배치되고 오버레이가 닫힘, isConfigured: true | |

## 5. 중간 노드 설정

> **v3 설계:** 중간 노드는 **ChoicePanel**에서 이전 노드의 outputDataType 기반 매핑 규칙으로 선택지를 제공한다. ServiceSelectionPanel은 사용하지 않는다. 최대 3단계: 처리 방식 → 액션 → 후속 설정.

### 5.1 중간 노드 추가 — ChoicePanel 진입

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 5.1.1 | "다음" placeholder 클릭 | **ChoicePanel** 오버레이 열림 (SSP가 아님) | |
| 5.1.2 | ChoicePanel 헤더 | 이전 노드의 출력 데이터 타입 라벨이 표시됨 (예: "단일 파일", "텍스트") | |
| 5.1.3 | SSP가 열리지 않는지 확인 | ServiceSelectionPanel은 시작/도착 placeholder에서만 열림 | |

### 5.2 Step 1 — 처리 방식 (목록형 데이터일 때만)

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 5.2.1 | 이전 노드 출력이 file-list인 경우 | "파일들을 어떻게 처리할까요?" 질문 + "한 파일씩" / "전체를 하나로 합쳐서" 선택지 | |
| 5.2.2 | 이전 노드 출력이 single-file인 경우 | 처리 방식 단계 생략, 바로 액션 목록 표시 | |
| 5.2.3 | "한 파일씩" 선택 | LOOP 노드 생성 → SINGLE_FILE 액션 목록 표시 (Step 2) | |
| 5.2.4 | "전체를 하나로 합쳐서" 선택 (FILE_LIST) | 노드 미생성, 현재 타입 유지 → actions 비어있으므로 완료 | |

### 5.3 Step 2 — 액션 선택

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 5.3.1 | 액션 목록 정렬 | priority 순으로 정렬되어 표시됨 | |
| 5.3.2 | "그대로 전달" (PASSTHROUGH) | 항상 목록 맨 아래에 위치 (priority: 99) | |
| 5.3.3 | follow_up 없는 액션 선택 (예: "긍정/부정 분류") | 노드 생성 → updateNodeConfig → ChoicePanel 닫힘 → 듀얼 패널 열림 | |
| 5.3.4 | follow_up 있는 액션 선택 (예: "내용 요약/정리") | 노드 생성 → Step 3(후속 설정)으로 전환 | |
| 5.3.5 | branch_config 있는 액션 선택 (예: "파일 종류별로 다르게 처리") | 노드 생성 → Step 3(분기 설정)으로 전환 | |
| 5.3.6 | "뒤로" 클릭 (처리 방식 거쳐 옴) | 배치된 노드 제거 → Step 1로 복귀 | |

### 5.4 Step 3 — 후속 설정

| # | 테스트 | 기대 결과 | 결과 |
|---|--------|-----------|------|
| 5.4.1 | follow_up 질문 표시 | "어떻게 요약할까요?" 등 질문 + 선택지 표시 | |
| 5.4.2 | branch_config 질문 표시 (multi_select) | 체크박스로 복수 선택 가능 | |
| 5.4.3 | type: "text_input" 옵션 | "직접 입력" 선택 시 자유 입력 필드 표시 | |
| 5.4.4 | 완료 | updateNodeConfig → ChoicePanel 닫힘 → 듀얼 패널 열림 | |
| 5.4.5 | "뒤로" 클릭 | 배치된 노드 제거 → Step 2(액션 선택)로 복귀 | |

## 테스트 진행 방법

1. **순서대로 진행**: 2장(캔버스) → 3장(Placeholder) → 4장(위자드) → 5장(중간 노드) → 6장(삭제) → 7장(닫기 규칙) → 8장(인터랙션) → 9장(Edge) → 10장(툴바) → 11장(라우팅)
2. **결과 기록**: 각 항목의 "결과" 칸에 `✅ Pass` / `❌ Fail (설명)` 기록
3. **Fail 발견 시**: 즉시 스크린샷 + 콘솔 에러 캡처
4. **매 섹션 완료 후**: 페이지 새로고침하여 상태 초기화 후 다음 섹션 진행
